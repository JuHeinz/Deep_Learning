Diskussion

Für das letzte Foto, (weißes Pferd), gab es über die Tage hinweg immer wieder andere Klassifikationen, die die höchste Confidence erzielen (wallaby, polar bear, llama etc). Zuerst ging ich davon aus, dass das Modell mehrmals täglich neu trainiert, aber trotzdem unter unveränderter Versionsnummer im CDN bereitgestellt wird.

Dann wurde mir jedoch klar, dass es wahrscheinlicher ist, dass mein Skalieren des Images im CSS zu einer veränderten Klassifikation führte. Hierdurch verändert sich der Input-Vektor, da weniger/mehr Pixel zur Verfügung stehen. Daraufhin wird folglich auch ein anderer Output-Vektor produziert.

Mich überrascht, dass das auf ImageNet trainierte Modell Menschen nicht erkennen kann, sondern stattdessen menschenähnliche Gegenstände (“mask", "wig") oder auch Kleidungsstücke wie "cardigan" ausgibt. Ich vermute deshalb, dass es keine Kategorie "human" im ImageNet Datensatz gibt.

Des Weiteren ist zu beobachten, dass es nicht immer klar ist, wann ein Bild als richtig klassifiziert gilt. Das Modell klassifiziert das vorletzte Foto als "pot, flowerpot", obwohl hier wohl jeder Mensch die Aufmerksamkeit auf die Früchte gerichtet hätte und sowohl "banana" als auch verschiedene Apfelsorten Teil des ImageNet Datensatzes sind.

Dokumentation

Diese Seite benutzt

  • den ImageClassifier von ml5 auf Basis des ImageNet-Datensatzes,
  • chart.js für das Plotten der Klassifikationen,
  • Bootstrap 5.3.5. als CSS-Framework
  • und lizenzfreie Bilder von Pixabay.

Bootstrap, ml5 und chart.js werden über Content Delivery Networks bezogen. Die Drag and Drop Funktion habe ich selbt implementiert.

Die Klassikfikation für die sechs voreingestellten Inputs wird bei Seitenaufruf (nach Triggern des DOMContentLoaded Events) automatisch durchgeführt. Um Probleme mit dem Erfüllen von Promises zu umgehen, verwende ich die then() Methode von JavaScript.

Die default Bar Chart von chart.js habe ich dahingehend angepasst, dass nun auch das Prozentzeichen in der y-Skala angezeigt wird. Die Confidence Werte, die vom Classifier ausgegeben werden, die ursprünglich zwischen 0 und 1 fallen, habe ich in Prozentwerte umgewandelt. Ebenso habe ich die Skala aller Diagramme so angepasst, dass 100% immer das Maximum darstellt. So können die Confidence-Werte über alle Inputs hinweg auf einen Blick verglichen werden.
